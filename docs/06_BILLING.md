# Биллинг, пакеты и списание уроков

## 1. Назначение модуля

Модуль биллинга отвечает за:
- покупку пакетов занятий
- хранение остатка уроков
- историю оплат
- историю списаний
- прогресс-бары пакетов
- задолженности
- правила списания при посещении и пропусках

---

## 2. Пакеты занятий

У ученика может быть один или несколько пакетов.

Пакет содержит:
- ID пакета
- ID ученика
- предмет
- тип занятий: individual / group
- всего занятий
- использовано занятий
- осталось занятий
- дата покупки
- стоимость
- статус оплаты

Пример:

```js
{
  id: 1,
  studentId: 101,
  subject: "Математика",
  lessonType: "individual",
  totalLessons: 16,
  usedLessons: 5,
  price: 24000,
  paid: true,
  purchasedAt: "2026-05-04"
}
```

## 3. Правила списания

Единое правило для всей системы. Никаких параллельных политик нет.

### 3.1 Урок СПИСЫВАЕТСЯ

1. Урок проведён (`status = done`).
2. Ученик не пришёл и не предупредил **минимум за 8 часов** до начала
   (`status = missed`, отмена/уведомление поступило позже, чем за 8 часов до `startAt`,
   либо не поступило вовсе).

### 3.2 Урок НЕ СПИСЫВАЕТСЯ

1. Отмена пришла **за 8 часов и более** до начала урока
   (`status = cancelled`, инициатор — `parent` или `student`).
2. Урок отменён **преподавателем или школой**
   (`status = cancelled`, инициатор — `teacher` или `school`),
   независимо от времени отмены.

### 3.3 Окно «8 часов»

- Точка отсчёта: `lesson.startAt`.
- Порог нестрогий: `now <= startAt - 8h` → «вовремя» (без списания).
- Сохраняем `lesson.cancelledAt` и `lesson.cancelledBy` для аудита.

```
              -8h            startAt
─────────────|───────────────|────────────────►
   вовремя        поздно
  (no charge)   (charge при missed)
```

## 4. Индивидуальные и групповые уроки

- **Индивидуальный (`lessonType: "individual"`):** один ученик, один `Charge` при списании.
- **Групповой (`lessonType: "group"`):** один урок, несколько участников.
  - Статус и решение о списании — **по каждому ученику отдельно**.
  - Для каждого, кто подпадает под правило списания, создаётся **свой `Charge`**
    в **свой пакет**.
  - Возможна ситуация: одного списали (опоздал с отменой), другого — нет
    (отменил вовремя), хотя урок один и тот же.

В сущности `Lesson` для групповых уроков:

```js
{
  id: 501,
  subject: "IT-академия: Python",
  teacherId: 7,
  lessonType: "group",
  startAt: "2026-05-12T17:00:00",
  participants: [
    { studentId: 101, status: "done" },
    { studentId: 102, status: "missed", cancelledAt: null },
    { studentId: 103, status: "cancelled", cancelledAt: "2026-05-11T22:00:00", cancelledBy: "parent" }
  ]
}
```

## 5. История списаний и Charge

Каждое списание = запись `Charge`:

```js
{
  id: 9001,
  lessonId: 501,
  packageId: 1,
  studentId: 101,
  amount: 1,
  reason: "lesson_done" | "lesson_missed_late",
  reasonText: "Урок списан: занятие проведено 12 мая, 17:00 — IT-академия: Python, преподаватель Иванова И.",
  createdAt: "2026-05-12T18:30:00"
}
```

`reasonText` готовится сразу при создании списания, чтобы UI просто рендерил строку.

### Эталонные формулировки `reasonText`

Эти формулировки фиксированы. Менять можно только при согласовании.

- `lesson_done`:
  «Урок списан: занятие проведено DD MMMM, HH:MM — Предмет, преподаватель И. Ф.»
- `lesson_missed_late`:
  «Урок списан: ученик не пришёл и отмена не была сделана минимум за 8 часов до начала занятия. (DD MMMM, HH:MM, Предмет)»

## 6. Что всегда видит родитель

В ЛК родителя обязательно доступно:

1. **Остаток занятий** по каждому пакету (число + прогресс-бар).
2. **История списаний** — лента всех `Charge` с фильтром по ребёнку и пакету.
3. **Причина каждого списания** — человекочитаемая строка (`reasonText`),
   начинается с «Урок списан: …» — пользователю не должно быть непонятно,
   почему списано.

### Спорных списаний в системе нет

В MVP функционал «оспорить списание» **отсутствует**:

- нет кнопки «Оспорить» в UI;
- нет сущности `disputes`;
- нет поля `disputed` у `Charge`;
- charge неизменяем после создания.

Спорные ситуации решаются вне системы (администратор/менеджер).
Поэтому правило 8 часов и формулировка `reasonText` должны быть
максимально понятны и предсказуемы.

## 7. Алгоритм списания (псевдокод)

```js
function onLessonStatusChange(lesson, newStatus, actor) {
  if (newStatus === "done") {
    chargeAffectedParticipants(lesson, "lesson_done");
    return;
  }

  if (newStatus === "cancelled") {
    // отмена не списывает никогда
    return;
  }

  if (newStatus === "missed") {
    chargeAffectedParticipants(lesson, "lesson_missed_late");
    return;
  }
}

function chargeAffectedParticipants(lesson, defaultReason) {
  for (const p of lesson.participants) {
    const reason = decideReason(lesson, p, defaultReason);
    if (!reason) continue;                     // вовремя отменил — пропуск
    createCharge({ lesson, studentId: p.studentId, reason });
    decrementPackageRemaining(p.studentId, lesson.subject);
    // notifyParent — удалено из MVP (см. 01_PRODUCT_ARCHITECTURE.md §6).
    // Сигнал родителю — через историю списаний и баннеры low/empty.
  }
}

function decideReason(lesson, p, defaultReason) {
  if (p.status === "cancelled") {
    if (p.cancelledBy === "teacher" || p.cancelledBy === "school") return null;
    const inTime = (lesson.startAt - p.cancelledAt) >= 8 * 3600 * 1000;
    return inTime ? null : "lesson_missed_late";
  }
  if (p.status === "done") return "lesson_done";
  if (p.status === "missed") {
    const inTime = p.cancelledAt && (lesson.startAt - p.cancelledAt) >= 8 * 3600 * 1000;
    return inTime ? null : "lesson_missed_late";
  }
  return null;
}
```

## 8. Граничные случаи

- **Преподаватель отменил за 1 час до урока** → не списывается (отмена школы > правила 8 часов).
- **Родитель отменил ровно за 8 часов (`>= 8h`)** → не списывается (порог нестрогий).
- **Группа: один пришёл, второй забыл** → одно списание (тому, кто забыл).
- **Остаток уже 0, а урок проведён** → `Charge` создаётся с пометкой «вне пакета»,
  уходит в задолженность; родителю — сигнал в интерфейсе (баннер «Долг по занятиям» на главной ЛК и запись в истории списаний).

## 9. UX-следствия (для `ux-designer`)

- На карточке пакета: остаток (`5 из 10`) + прогресс-бар + цвет по остатку
  (зелёный → жёлтый → красный при `≤2`).
- В истории списаний — иконка причины:
  - ✓ зелёная — проведён;
  - ⚠ жёлтая — пропуск, отмена не была сделана минимум за 8 часов до начала занятия.
- В плашке «Урок завтра» на главной ЛК родителя — подсказка с конкретным временем:
  «Чтобы отменить без списания, успейте до HH:MM».
- Групповой урок отображается **одной карточкой** со списком участников и индивидуальными статусами.
