```
┌──┐  блок / контейнер              [ Кнопка ]   primary CTA
```

## A2. Lead-модалка (фактическая, одна универсальная)

Открывается при клике на любую CTA: hero «Записаться», карточка направления «Записаться», карточка локации «Связаться».

```
┌────────────────────────────────────┐
│ Запись на занятие              [✕] │
│ ✓ Пробное занятие — бесплатно      │
│ Оставьте контакты — свяжемся в час │
├────────────────────────────────────┤
│ Имя        ____________            │
│ Телефон    +7 ___ ___ __ __        │
│ Направление *                      │
│ [ — выберите ▾ ]                   │
│   • Школа 1–11 классы / репетиторы │
│   • IT-Академия                    │
│   • ОГЭ / ЕГЭ                      │
│ Предмет / комментарий              │
│ ____________________________       │
│                                    │
│ [ Отправить заявку ]               │
└────────────────────────────────────┘
```

**Поведение:**
- Имя, телефон, направление — обязательны. Без направления показывается «Выберите направление.».
- Предзаполнение `direction` зависит от точки входа: hero — пусто; карточки направлений — заполнено `school` / `academy` / `exam`; «Связаться» в локации — пусто.
- На `submit` вызывается `window.XSchool.addLead({ name, phone, mode, direction, subject, source: 'landing' })`. Поля `mode` и `direction` хранят одно значение — для обратной совместимости с админкой.
- После успеха — плашка «Заявка сохранена. Мы свяжемся с вами в ближайшее время.»
- Никаких сущностей `Parent`/`Student`/`Lesson` форма НЕ создаёт. Дальнейшую работу с заявкой ведёт админ/директор в `admin.html → Люди → Заявки`.

# F. Кросс-экранные элементы

## F1. Карточка пакета (универсальная)

Используется на главной родителя, в карточке ребёнка, при покупке.

```
┌─────────────────────────────────────────────┐
│ Математика · Настя · индивидуально          │
│                                             │
│ ▮▮▮▮▮▯▯▯▯▯       5 из 10                    │
│ 5 уроков осталось                           │
│                                             │
│ < История >    [ Продлить ]                 │
└─────────────────────────────────────────────┘
```

Цвет полосы прогресса:
- зелёный — `remaining > 30%`
- жёлтый — `remaining ≤ 30%`
- красный — `remaining ≤ 2`

## F2. Уведомление «Урок завтра» — УДАЛЕНО ИЗ MVP

В MVP отдельного inbox нет. На «Главной» родителя/ученика остаётся
data-driven плашка завтрашнего урока с конкретным дедлайном отмены
без списания (вычисляется из `lesson.startAt − 8h`).

## F3. Карточка группового урока

Унифицированный вид для расписания родителя/ученика и CRM преподавателя.

```
┌─────────────────────────────────────────────┐
│ ГРУППА · IT-академия: Python · 18:30        │
│ Преподаватель: Иванова И.                   │
│                                             │
│ Участники (6)                               │
│  • Артём К.   ⚪                             │
│  • Маша Л.    ⚪                             │
│  • Дима С.    ✕ отменил (вовремя)           │
│  • Соня В.    ⚠ ожидается, без отмены       │
│  ···                                        │
│                                             │
│ [ Открыть ]                                 │
└─────────────────────────────────────────────┘
```

В ЛК ученика: подсвечивается только сам ученик; остальные — серым/обезличенно («ещё 5 учеников»).
В ЛК родителя: подсвечиваются свои дети, остальные — обезличенно.

## F4. Пустые состояния

- Дети у родителя: «Пока никого не записано. [ Записать ребёнка ]»
- Расписание: «На этой неделе уроков нет. < Перейти к ближайшему >»
- ДЗ: «ДЗ ещё нет. Начнём после первого урока 🙂»
- Списания: «История пуста — пакетов ещё не было.»

## F5. Состояния ошибок

- Сетевая (заглушка для будущего бэка): toast «Не удалось сохранить. Попробуйте ещё раз».
- Низкий остаток (`remaining ≤ 2`): жёлтый бейдж «⚠ Низкий остаток» + CTA «Продлить».
- Нулевой остаток (`remaining = 0`): красный баннер сверху ЛК родителя «Пакет закончился — продлите, чтобы не пропускать уроки».

# H. Принципы UX

1. **Один главный CTA на экран.** Остальные действия — secondary.
2. **Деньги — всегда явно.** Никаких скрытых списаний; в каждом уведомлении про урок видно дедлайн отмены.
3. **Mobile-first.** Каждый экран сначала рисуется под 360–414px, потом масштабируется.
4. **Темы режимов.** Публичный сайт переключает темы; ЛК наследует **базовую тёмную тему** + один акцент в зависимости от того, какой режим активен у выбранного ребёнка/предмета (ровно один акцент в момент времени, чтобы не пестрить).
5. **Групповые уроки = одна карточка.** Никогда не дублируем урок на каждого ученика в расписании родителя; внутри карточки — статусы участников.
6. **Прогресс — визуально.** Везде, где есть «X из Y», обязательно прогресс-бар.
7. **Время отмены — конкретно.** Не «за 8 часов», а «до 09:00 13 мая».
