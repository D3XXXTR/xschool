### Сущность Service
Управляемый слой над Subject. Один service ↔ один subject через общий `id`.
```

  packages:[{lessons, discount}], createdAt, updatedAt }
```

- `state.services[]` в seeds (math/english/it/ege-math).
- Миграция: для всех существующих subjects, у которых нет service, создаются
  записи с разумными дефолтами (BASE_PRICE для индивидуального, ~55% для группы,
  пакеты 4/8/16 со скидками 0/6/12 %).
- Lookups: `getService(id)`, `getServiceBySubject(id)`, `activeServices()`.
- Write-API:
  `addService` (создаёт service + subject если нужно, дедуп по id),
  `updateService` (синхронизирует title/category в subject),
  `toggleServiceActive`, `updateServicePrice`, `updateServicePackage`.

  - использует `sizesFor` вместо хардкода `SIZES`;
