# 04 — Роли пользователей

В MVP пять ролей. Все права прописаны на стороне UI и проверяются гардом
`XAuth.requireSession(role)`.

## Гость

- Видит публичный сайт `index.html`: header, hero, направления, экспертов, отзывы, локацию, footer.
- **Не переключает режимы** — публичная страница без `data-mode`, направления показаны как обычные карточки.
- Может оставить **lead-заявку** через форму на главной (`XSchool.addLead`).
- Поля заявки: имя, телефон, направление (`school`/`academy`/`exam`), предмет/комментарий.
- Не видит цены, пакеты, расписание, ДЗ, статистику, личные данные учеников/родителей/преподавателей.

## Родитель

- Связан с детьми по `phone`.
- Видит: всех своих детей, расписание, ДЗ, пакеты, оплаты, историю списаний и платежей, статистику.
- Может: оплатить пакет (мастер из 5 шагов), отменить или перенести урок, открыть карточку ребёнка.
- **Не меняет правила биллинга** — действует единое правило 8 часов из `06_BILLING.md`. Никаких индивидуальных policy у пакетов.
- Не может: проводить уроки, выдавать/проверять ДЗ.

## Ученик

- Видит только себя.
- Видит: ближайший урок, расписание, свои ДЗ, остатки пакетов (без сумм), прогресс.
- Может: сдавать ДЗ — текст + до 5 файлов (jpg/png/pdf, ≤10 МБ/файл, ≤50 МБ всего).
- При первом открытии `assigned`-ДЗ автоматически переходит в `in_progress`.
- Может пересдать после `needs_revision`.
- Не может: видеть платежи и других учеников.

## Преподаватель

- Видит: своих учеников, своё расписание, свои уроки и ДЗ.
- Может: создавать уроки (индивидуальные и групповые), проводить уроки и ставить статусы участников, выдавать и проверять ДЗ, оставлять комментарии и оценки.
- Не может: видеть оплаты родителей. Видит только остаток пакета по ученику для контекста.

## Администратор / Директор (в MVP)

- Реализован в `admin.html`. Демо-вход `+79990000000 / 0000`.
- Sidebar: Дашборд, Люди, Финансы, Обучение, Настройки.
- Функционал:
  - **Люди:** ученики, родители, преподаватели, назначения, заявки.
  - **Финансы:** оплаты, задолженности, пакеты, списания.
  - **Обучение:** расписание, домашние задания.
  - **Настройки:** тарифы и услуги (включить/отключить, цены, скидки пакетов).
- Может: добавлять и редактировать пользователей, обрабатывать заявки, создавать уроки и пакеты вручную, управлять услугами, изменять статус оплат.
- Не имеет права удалять пользователей и платежи (только `active=false` для услуг). Charges не оспариваются (см. `06_BILLING.md` §6).

## Связки

- Родитель ↔ ученик: по `phone` родителя, указанному при создании ученика администратором.
- Преподаватель ↔ ученик: через `Assignment` (создаёт админ) и `Lesson` (создаёт преподаватель или админ).

## Жизненные циклы

### Lead → Student
1. Гость заполняет форму на лендинге → `XSchool.addLead({...})` → `Lead{ status: 'new' }`.
2. Админ открывает заявку, может: сменить статус (`new → in_progress → booked / rejected`), создать `Parent`, создать `Student` (привязка по phone родителя).
3. Дальнейшее назначение преподавателя и создание `Lesson` — отдельные шаги в админке/CRM.

### Homework status
`assigned → in_progress → submitted → checked | needs_revision`. Финал — `checked`. После `needs_revision` ученик может вернуться в `in_progress` или сразу `submitted`.

(Старое имя `needs_rework` устарело — используется `needs_revision`.)
